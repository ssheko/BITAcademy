Fail-Over 기능: 한 서버에서 프로그램이 오류나 죽어버려도 옆 서버에서 바로 대신할 수 있도록.
SOAP (Simple Object Access Protocol) 아키텍처: ex) 버스, 지하철 안내 앱
로드밸런싱: 같은 서버 여러대 중 한 서버에만 트래픽이 몰리지 않도록 부하가 적은 서버에게 요청을 하도록 부하를 분산하는 서버를 운영
Dedicated 서버: 특정 요청만 받는 서버
마샬링 (marshalling): 두 장비간의 네트워크 통신시 데이터 변환. 전송에 적합한 다른 데이터 형식으로 변환. 통신 단계가 많을수록 마샬링이 많이 발생. 이것도 부하가 많이 발생.

특정한 이유에서만 Dedicated 모델을 사용하지만 일반적인 경우 Web Farm을 통한 Scale-out을 권장.
IPC: 프로세스끼리 통신. 외부가 아닌 프로세스끼리 통신하면 더 빠름.
RPC의 속도가 매우 느리기 때문에 참고만. (RPC 프로토콜 알 필요 x)
